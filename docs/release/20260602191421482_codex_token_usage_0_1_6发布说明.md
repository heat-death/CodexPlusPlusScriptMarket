# Codex Token Usage 0.1.6 发布说明

生成时间：2026-06-02 19:14

## 版本

- 插件：`Codex Token Usage`
- 版本：`0.1.6`

## 本次修复

### 1. 账本优先聚合

展示层不再主要依赖易丢失的即时 `currentTurn` 缓存，而是优先从账本事件重新派生：

- 当前会话最新一轮统计
- 当前会话多轮历史
- 每轮调用次数与调用合计

### 2. 修复调用次数长期为 1

补齐“同一轮内多次调用”的账本归并逻辑：

- 强匹配优先用 `requestId/responseId/callId/eventId`
- 跨来源重复采集再做窗口去重
- context-only 事件只补上下文，不再覆盖调用明细

### 3. 修复会话切换串统计

补齐同项目不同会话、同会话不同项目、同会话不同轮次的隔离：

- 同项目不同会话隔离
- 同会话不同项目隔离
- 同会话不同轮次隔离
- 展示层仍只显示当前会话最新一轮

### 4. 新增重启后历史恢复

当程序重启后页面内无缓存时，插件会尝试通过宿主 bridge 读取当前会话的历史 rollout 记录，动态恢复：

- 历史调用次数
- 本轮调用合计
- 输入输出
- 缓存命中
- 上下文用量

## 显示口径

文案统一为：

`本轮调用合计 · 输入 · 输出 · 缓存读 · 上下文 · 调用 N 次 · 耗时`

其中：

- `本轮调用合计` 表示当前轮次内所有调用的合计
- `上下文` 表示当前最新可信上下文读数
- 若 `totalTokens` 为估算值，会标记 `(估算)`

## 验证结果

已通过：

- `npm test`
- `npm run check:index`

## 兼容说明

“重启后无缓存自动恢复历史统计” 依赖 `CodexPlusPlus` 宿主侧同步支持。如果宿主未更新到包含 `thread-usage-history` bridge 的版本，则插件会继续保持现有实时统计能力，但历史恢复不会生效。
